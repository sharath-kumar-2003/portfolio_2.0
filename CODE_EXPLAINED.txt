--- PROJECTS SECTION — 3D CAROUSEL (Lines 197-310) ---

  ★ THIS IS THE STAR FEATURE — inspired by the Lun Dev video ★

  <div class="carousel-scene">        → The 3D "stage" (has perspective)
    <div class="carousel-slider">      → The rotating ring (has preserve-3d)
      <div class="carousel-item">      → Each project card

  How the 3D works:

  1. carousel-scene has perspective: 1200px
     This creates a "camera" at a certain distance. Objects closer appear
     bigger, objects farther appear smaller — just like real life.

  2. carousel-slider has transform-style: preserve-3d
     This tells the browser: "don't flatten the children into 2D.
     Let them actually exist in 3D space."

  3. Each carousel-item is positioned using:
     transform: rotateY(angle) translateZ(420px)

     Imagine standing in the center of a room:
     - rotateY spins the card to face a certain direction (like a clock)
     - translateZ pushes it outward from the center (the room's radius)

     With 5 cards and 360°, each card is 72° apart:
       Card 1: rotateY(0°)    translateZ(420px)  → directly in front
       Card 2: rotateY(72°)   translateZ(420px)  → to the right
       Card 3: rotateY(144°)  translateZ(420px)  → behind-right
       Card 4: rotateY(216°)  translateZ(420px)  → behind-left
       Card 5: rotateY(288°)  translateZ(420px)  → to the left

  CSS variables used:
    --quantity: 5          → total number of cards
    --position: 1 to 5    → which position this card is at
    --tz: 420px           → how far from center (the radius)
    --carousel-angle: 0deg → current rotation (JS updates this)

  Each card contains:
    carousel-top:    Icon + number (01, 02, etc.)
    carousel-body:   Title + description
    carousel-footer: Tech tags + arrow button

  Controls:
    carousel-btn (prev/next): Arrow buttons below the carousel
    carousel-dots:            5 dots showing which card is "front"
    carousel-reflection:      Subtle glow underneath for depth illusion

================================================================================
  FILE 2: styles.css — THE STYLING
================================================================================

--- CSS VARIABLES (:root) ---

  Variables are reusable values. Instead of typing #8b5cf6 everywhere,
  you write var(--accent-violet). Change it once, updates everywhere.

  --bg: #000000              → Pure black (OLED friendly)
  --surface-1, 2, 3          → Slightly lighter blacks for cards/elements
  --border: rgba(255,255,255,0.04) → Nearly invisible white borders
  --text-primary: #f0f0f5    → Almost white (easier on eyes than pure #fff)
  --text-secondary: #8888a0  → Medium gray for descriptions
  --text-muted: #44445a      → Dark gray for subtle text
  --accent-violet: #8b5cf6   → Your primary accent color
  --ease-out-expo            → A smooth deceleration curve for animations
  --ease-spring              → A bounce/overshoot curve


--- KEY CSS CONCEPTS USED ---

  1. backdrop-filter: blur(20px)
     Makes the background behind an element blurry — this is "glassmorphism."
     The card itself is semi-transparent, so you see a frosted glass effect.

  2. perspective & transform-style: preserve-3d
     Already explained above in the carousel section.

  3. @property --border-angle
     This is a *registered CSS custom property*. Normally you can't animate
     CSS variables, but @property lets you define the type (<angle>) so
     the browser knows HOW to animate it. Used for the rotating border.

  4. conic-gradient (on bento-card::before)
     A gradient that sweeps in a circle, like a radar. Combined with
     mask-composite: exclude, it creates an animated border-only effect:
     - The ::before covers the whole card
     - The mask cuts out the inside, leaving only the 1px border visible
     - The conic-gradient rotates, making the border "shine" around

  5. will-change: transform
     Tells the browser "this element will animate soon" so it can
     optimize by putting it on its own GPU layer. Improves performance.

  6. clamp(min, preferred, max)
     Responsive sizing without media queries.
     Example: font-size: clamp(3rem, 8vw, 7rem)
     → At minimum: 3rem (48px)
     → Preferred: 8% of viewport width
     → At maximum: 7rem (112px)

  7. CSS Animations (@keyframes)
     
     orbFloat:    Orbs drift and scale up/down over 25 seconds
     marquee:     Slides text infinitely to the left
     charReveal:  Each letter slides up from below + slight rotation
     fadeSlideUp: Elements fade in while moving upward
     borderRotate: Rotates the --border-angle for animated borders
     pulse:       Green availability dot pulses outward
     ringPulse:   Identity card rings breathe in and out
     scrollDown:  The scroll hint line has a light moving downward
     dotGlow:     Marquee diamonds glow and dim

  8. Responsive (@media queries)
     
     @media (max-width: 1024px): Tablet adjustments (smaller carousel)
     @media (max-width: 768px):  Mobile (hamburger menu, stack layout)
     @media (max-width: 480px):  Small phones (even smaller text)


--- 3D CAROUSEL CSS (The Complex Part) ---

  .carousel-scene {
      perspective: 1200px;  → Camera distance — smaller = more dramatic 3D
  }

  .carousel-slider {
      transform-style: preserve-3d;  → Children live in 3D space
      transform: rotateY(var(--carousel-angle));  → JS changes this to rotate
      transition: transform 0.8s;  → Smooth rotation animation
  }

  .carousel-item {
      position: absolute;  → All cards stacked on top of each other
      left: 50%; top: 50%; → Centered
      margin-left: -160px; → Offset by half width (320/2) to truly center
      transform: 
          rotateY(calc(360deg / var(--quantity) * (var(--position) - 1)))
          translateZ(var(--tz, 420px));
      
      The calc() formula:
        360° ÷ 5 cards = 72° per card
        Card 1: (1-1) × 72 = 0°
        Card 2: (2-1) × 72 = 72°
        Card 3: (3-1) × 72 = 144°
        ... and so on
  }

  When JS changes --carousel-angle on the slider, ALL cards rotate together
  because they're children of the slider. Since each card has its own 
  rotateY offset, the whole ring turns like a lazy susan.



================================================================================
  FILE 3: main.js — THE INTERACTIVITY
================================================================================

Everything is wrapped in DOMContentLoaded — this fires when the HTML is 
fully parsed (but images might still be loading). This ensures all elements
exist before we try to access them.


--- PAGE LOADER ---

  window.addEventListener('load', ...) fires when EVERYTHING is loaded.
  After 1 second delay, we add class "hidden" to the loader div.
  CSS transitions opacity to 0 and sets visibility: hidden.


--- NAVBAR SCROLL ---

  On scroll, if scrollY > 60px, add class "scrolled" to the nav.
  CSS then: shrinks height from 80px→60px, darkens background.


--- HAMBURGER MENU ---

  Click toggles "active" class on hamburger (CSS animates 3 lines → X)
  and "open" class on mobile menu (CSS fades in the overlay).
  document.body.style.overflow = 'hidden' prevents scrolling while open.


--- SCROLL PROGRESS ---

  scrollTop: how far you've scrolled from top
  scrollHeight: total page height
  clientHeight: visible window height
  
  Formula: (scrollTop / (scrollHeight - clientHeight)) × 100 = percentage
  Set that as the width of the progress bar.


--- SPLIT TEXT ENGINE ---

  This finds the hero title, reads each line's text, then:
  1. Saves the text from data-text attribute
  2. Clears the element
  3. Creates a <span class="char"> for each character
  4. Sets animation-delay: 0.6s + (charIndex × 0.035s)
     So "E" starts at 0.6s, "n" at 0.635s, "g" at 0.67s, etc.
  5. CSS @keyframes charReveal slides each span up from 110% below

  The result: letters appear to "type" or "flip" into place one by one.


--- STAT COUNTER ---

  Uses requestAnimationFrame for smooth 60fps animation.
  
  Easing formula: 1 - Math.pow(2, -10 * progress)
  This is "ease-out exponential" — starts fast, slows down at the end.
  The number appears to "decelerate" as it approaches the target.
  
  IntersectionObserver triggers it only when the stats row is 50% visible.
  statsCounted flag ensures it only runs once.


--- SCROLL REVEAL (IntersectionObserver) ---

  IntersectionObserver is the modern way to detect when elements enter
  the viewport. Much better than scroll event listeners (performance).

  Options:
    threshold: 0.06  → Trigger when 6% of element is visible
    rootMargin: '0px 0px -60px 0px'  → Shrink detection area by 60px
                                        from bottom (elements must scroll 
                                        a bit more before revealing)

  When triggered: reads data-delay, waits that many ms, adds class "active".
  CSS transitions from opacity:0 + translateY(40px) to opacity:1 + translateY(0).


--- ACTIVE NAV PILL ---

  Another IntersectionObserver watches all sections.
  When a section becomes 30% visible, it finds the matching nav link
  and moves the pill underneath it.

  updateNavPill() gets the link's position relative to its parent,
  then sets the pill's left offset and width to match.

  On hover: pill follows the hovered link.
  On leave: pill snaps back to the actual active section's link.


--- EMAIL COPY ---

  Prevents default link behavior (no mailto: popup).
  Uses navigator.clipboard.writeText() — modern async clipboard API.
  Temporarily changes the text to "✓ Copied!" with green color.
  After 2 seconds, restores original text.


--- ★ 3D CAROUSEL CONTROLLER ★ ---

  Variables:
    quantity:     Number of cards (5)
    angleStep:    360 / 5 = 72° per step
    currentAngle: Current rotation in degrees
    currentIndex: Which card is "front" (0-4)

  rotateCarousel(angle, index):
    Sets --carousel-angle CSS variable on the slider element.
    CSS transition handles the smooth animation.
    Updates dot indicators.

  Auto-Rotate:
    setInterval runs every 4000ms (4 seconds).
    Each tick subtracts one angleStep (72°) = rotates one card forward.
    Pauses when mouse enters the carousel scene.
    Resumes when mouse leaves.

  Button Controls:
    Prev: adds angleStep (rotates backward)
    Next: subtracts angleStep (rotates forward)
    Both reset the auto-rotate timer.

  Dot Controls:
    Calculates the difference between target and current index.
    Multiplies by angleStep to get the exact rotation needed.

  Keyboard:
    ArrowLeft → same as Prev button
    ArrowRight → same as Next button

  Drag to Rotate:
    mousedown: Records starting X position and current angle.
    mousemove: Calculates horizontal distance dragged.
               Multiplies by 0.3 to reduce sensitivity.
               Updates carousel angle in real-time.
    mouseup:   Snaps to nearest card position using Math.round().
               This prevents the carousel from stopping between cards.

  Touch Support:
    Same logic as drag, but using touchstart/touchmove/touchend events.
    { passive: true } tells the browser we won't call preventDefault(),
    which allows smoother scrolling on mobile.


--- MAGNETIC BUTTONS ---

  On mousemove over a .magnetic element:
    Calculates cursor position relative to button center.
    Moves button toward cursor: translate(x * 0.25, y * 0.25)
    The 0.25 factor means button moves 25% of the cursor distance.

  On mouseleave:
    Resets transform to '' (original position).
    Adds a spring transition for a satisfying "snap back" effect.
    Removes the transition after 500ms so future moves aren't delayed.

  On click:
    Sets --ripple-x and --ripple-y CSS variables to click position.
    Adds class "ripple" which triggers CSS ::after expanding radial gradient.
    Removes after 600ms.


--- 3D CARD TILT ---

  Elements with data-tilt attribute get this treatment.

  On mousemove:
    x, y = cursor position normalized to [-0.5, 0.5] range
    transform: perspective(800px) rotateX(-y*6) rotateY(x*6) translateZ(8px)
    
    -y*6 for rotateX: moving mouse up tilts card toward you (negative Y = up)
    x*6 for rotateY: moving mouse right tilts card right
    translateZ(8px): slight "lift" effect

    Also sets --glare-x and --glare-y for the CSS radial-gradient shine.

  On mouseleave: smooth reset with ease-out transition.
  On mouseenter: removes transition so movement is instant (responsive).


--- PARTICLE CANVAS ---

  Creates 80 particles (or fewer on small screens).
  
  Each particle has:
    x, y:      Position
    vx, vy:    Velocity (speed + direction)
    radius:    Size (0.3 to 2.1 px)
    opacity:   Transparency (0.1 to 0.6)
    hue:       Color (258=violet or 220=blue)

  Every frame (60fps via requestAnimationFrame):
    1. Clear the canvas
    2. For each particle:
       - Check mouse distance. If within 150px, push particle away.
       - Apply damping (multiply velocity by 0.992 to slow down)
       - Move by velocity
       - Bounce off edges
    3. Draw connections:
       - For every pair of particles, if distance < 180px, 
         draw a faint line between them. Closer = more visible.
    4. Draw particles:
       - Outer glow: larger circle, very transparent (ambient halo)
       - Core: actual particle dot

  Mouse repulsion formula:
    force = (150 - distance) / 150
    Closer mouse = stronger push. At 0px distance, force = 1 (max).
    At 150px, force = 0 (no effect). Smooth falloff.


--- PARALLAX DEPTH ---

  On mousemove, each background orb shifts position based on mouse.
  depth = (orbIndex + 1) * 15
  
  Orb 1 shifts 15px per half-screen of mouse movement.
  Orb 2 shifts 30px.
  Orb 3 shifts 45px.
  
  This creates depth: closer orbs move more, distant ones move less.
  Same principle as looking out a car window — nearby trees fly by,
  distant mountains barely move.

  Only active on screens wider than 768px (disabled on mobile for perf).


--- SECTION HEADER PARALLAX ---

  On scroll, section headers get a subtle translateY offset.
  progress = how far the header has traveled through the viewport.
  Creates a "floating" effect as you scroll past sections.

  .reveal-stagger   → Children fade in with sequential delays
  .reveal-scale     → Scale up from 0.9 to 1.0 on scroll

  [data-count]      → Target number for count-up animation
  .active           → Applied by JS when element should be visible
  .scrolled         → Applied to nav when page is scrolled
  .is-active        → Applied to nav link for current section
  .hidden           → Applied to loader when page is loaded


================================================================================
  HOW TO MODIFY THINGS
================================================================================

  Change colors:      Edit :root variables in styles.css (lines 6-21)
  Change fonts:       Edit Google Fonts link in HTML + :root variables
  Add a project:      Copy a carousel-item div, change --position and 
                      --quantity. Add a dot to carousel-dots.
  Change carousel speed:  In main.js, find setInterval → change 4000 
                         (milliseconds). 3000 = faster, 6000 = slower.
  Change carousel size:   In styles.css, edit --tz value (420px default).
                         Larger = wider ring, smaller = tighter ring.
  Change tilt intensity:  In main.js, find "* 6" in the tilt section.
                         Higher = more dramatic tilt.
  Change particle count:  In main.js, find "Math.min(80, ..." and change 80.
  Add new section:        Create HTML section, add "reveal" class,
                         add a nav link + update JS section observer.
